# Gemini Companion Starter (Gemini CLI 个人助理初始化系统)

[English](./README.md)

这是一段专为 Gemini CLI 设计的“初始化提示词 (Bootstrap Prompt)”，旨在将其从一个“一次性问答工具”，改造为具备长期协作记忆和个性化工作流的**专属个人助理**。

## 这个项目解决什么问题

很多“个人助理型提示词”虽然写得很完整，但不一定适合 CLI 的真实执行方式，常见的问题包括：
- 过度依赖聊天记录，一刷新就失忆
- 每次对话试图读取过多文件，导致响应慢和 Token 浪费
- “状态追踪”容易破裂，或者缺乏文件系统实体操作导致幻觉

这套 **Gemini Companion Starter** 的核心思路是：将文件系统用作 Gemini 的“外部记忆层”。它借鉴了先进的 Agent 记忆系统（类似 OpenClaw），把 daily log 和长期 memory 设置为工作区中的明确资产，让 Gemini 少一点“每次重新认识你”的工具感，多一点“持续协作”的连续性。

### 它改变了什么

- 你的偏好、表达规则和项目决策不再只散落在聊天记录里。
- 临时信息（如每日想法）和长期信息分开存储。
- Gemini 重新进入你的工作区时会更有连续性，不再依赖你反复重申。

### 为什么会显得更“人性化”

如果只有原始的工具，你通常要反复重讲：
- 你希望它怎么组织答案
- 你在这个工作区里是什么身份
- 这个项目之前已经做过哪些重要的取舍
- 上一次工作停在什么位置

通过这段提示词，所有的信息被分层存放到了 `.assistant/STYLE.md`、`.assistant/WORKFLOW.md`、`.assistant/MEMORY.md`、`memory/projects/*.md` 这样的本地结构中。它们变成了**可被编辑、可被审计、可持续演化**的项目资产。

### 具体例子

1. **表达风格不再每次重讲：** 比如“先给结论、再讲风险、保持简洁”，这些偏好会固化到 `STYLE.md` 中。
2. **项目决策不会轻易重置：** 比如“当前阶段先实现核心流程”，放入 `memory/projects/architecture.md`，避免下次从零讨论。
3. **临时信息不污染长期记忆：** 尚未确认的接口文档、今天的碎碎念，归入 `memory/daily/YYYY-MM-DD.md`。

## 它的工程优势 (核心特性)

1. **记忆懒加载 (Lazy Loading)：** 默认仅读取少量的必须规则文件（如 `SYSTEM.md`），真正需要上下文时再按需读取具体项目信息，极大降低了工具调用风暴与 Token 花销。
2. **实体级状态判定：** 不采用易错的状态文件，直接通过 `USER.md` 和 `STYLE.md` 等文件是否有实质内容（>3行）判断是否需要进入 Bootstrap 向导提问模式。
3. **防呆与抗幻觉执行：** 在提示词层面严格约束了大模型的行为，强制要求它必须调用底层的系统工具真实地去创建、读写文件，严禁只在对话框内发大段文件代码示例。
4. **归档触发器：** 摒弃模糊的任务结束判定。必须由用户明确发送如 `"/done"`、`"结束"` 或 `"总结会话"` 等主动指令，才会复写 `last-session.md` 以沉淀当日记忆。

## 快速开始

1. 确保你已安装并配置好具备文件操作能力的 Gemini CLI。
2. 打开你想初始化的项目终端工作区：`cd /path/to/your/project`
3. 复制本项目中的 [Gemini-Companion-Starter.md](./Gemini-Companion-Starter.md) 的所有文本。
4. 在 CLI 会话中一次性发给 Gemini。
5. 顺其自然地跟随向导，回答它的 1-3 个简短问题。
6. 初始化完成，开启你的贴身助理协作之旅吧！
